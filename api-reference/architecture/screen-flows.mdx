## Screen-by-Screen Breakdown

### 1. Onboarding (Profile Setup)

The driver enters Aarokya from NammaYatri for the first time. NammaYatri passes the driver's identity via the SDK token flow.

```mermaid
sequenceDiagram
    actor Driver
    participant App as Aarokya SDK
    participant API as Aarokya Backend

    Note over Driver,App: First-time entry from NammaYatri
    Driver->>App: Enters Name, DOB, Gender, Email, Mobile
    Driver->>App: Taps "Submit"
    App->>API: POST /user/profile (create/update)
    API-->>App: 200 OK (profile saved)
    App->>Driver: Navigate to Add Dependants
```

**Screen fields:**
- Name (pre-filled from NammaYatri if available)
- Date of Birth
- Gender
- Email
- Mobile (pre-filled, read-only)

**APIs called:**

| API | Method | Purpose |
|-----|--------|---------|
| `POST /user/profile` | Create | Create user profile with onboarding details |

### 2. Add Dependants

Driver adds family members who will be covered under the insurance plan. Can be skipped.

```mermaid
sequenceDiagram
    actor Driver
    participant App as Aarokya SDK
    participant API as Aarokya Backend

    Driver->>App: Add Family Member 1 (name, DOB, relationship)
    Driver->>App: Add Family Member 2
    Driver->>App: Add Family Member 3
    Driver->>App: Taps "Submit" (or "Skip")
    App->>API: POST /user/dependants (batch create)
    API-->>App: 200 OK (dependants saved)
    App->>Driver: Navigate to Available Benefits
```

**Screen fields (per dependant):**
- Name
- Date of Birth
- Gender
- Relationship (spouse, child, parent, etc.)
- Add (+) button for more members

**APIs called:**

| API | Method | Purpose |
|-----|--------|---------|
| `POST /user/dependants` | Create | Save family members as dependants |

### 3. Available Benefits

Shows the benefits the driver can avail -- free doctor consultations, insurance plans, etc.

```mermaid

    participant App as Aarokya SDK
    participant API as Aarokya Backend
```

**Screen layout:**
- List/grid of benefit cards
- Each card: title, short description, icon
- Examples: "Free Doctor Consultations", "Health Insurance"

**APIs called:**

| API | Method | Purpose |
|-----|--------|---------|
| `GET /benefits/available` | Read | List benefits available to this driver |

### 5. Insurance Enrollment

The main enrollment screen. Shows premium amount, lets the driver add/remove dependants from coverage, and proceeds to payment.

```mermaid
sequenceDiagram
    actor Driver
    participant App as Aarokya SDK
    participant API as Aarokya Backend
    participant PG as Hyper PG

    App->>API: GET /user/dependants
    API-->>App: List of dependants
    App->>API: POST /insurance/premium/compute
    Note over API: Calculate premium based on<br/>self + selected dependants
    API-->>App: Premium amount
    Driver->>App: Select/deselect dependants (+/-)
    App->>API: POST /insurance/premium/compute (recalculate)
    API-->>App: Updated premium
    Driver->>App: Taps "Proceed to Pay"
    App->>API: POST /insurance/enroll
    API-->>App: Enrollment initiated, redirect to payment
    App->>PG: Open Hyper PG (setup autopay)
```

**Screen fields:**
- Premium amount (computed dynamically)
- Dependant list with +/- toggle
- "Setup Autopay" button
- Proceeds to Hyper PG for payment

**APIs called:**

| API | Method | Purpose |
|-----|--------|---------|
| `GET /user/dependants` | Read | Fetch saved dependants for selection |
| `POST /insurance/premium/compute` | Compute | Calculate premium for self + selected dependants |
| `POST /insurance/enroll` | Create | Initiate benefit enrollment |

### 6. Payments Page (UPI / Mandate Setup)

Payment collection via Hyper PG. Supports UPI payment and mandate setup for recurring autopay.

```mermaid
sequenceDiagram
    actor Driver
    participant App as Aarokya SDK
    participant PG as Hyper PG
    participant UPI as UPI App (GPay/PhonePe/etc.)
    participant API as Aarokya Backend

    App->>PG: Open payment page
    Driver->>PG: Selects UPI / Mandate Setup
    PG->>UPI: Redirect to UPI app
    Driver->>UPI: Authorizes payment
    UPI-->>PG: Payment success
    PG-->>App: Payment callback (success)
    App->>API: POST /mandate/create
    API-->>App: Mandate created
    App->>Driver: Show success screen
```

**Screen layout:**
- UPI payment option
- Mandate setup option
- Payment app icons (GPay, PhonePe, Paytm)
- Amount display

**APIs called:**

| API | Method | Purpose |
|-----|--------|---------|
| `POST /mandate/create` | Create | Create autopay mandate for recurring deductions |

### 8. Wallet Creation

Wallet setup screen. Collects any additional info needed for the health savings wallet (backed by Transcorp).

```mermaid
sequenceDiagram
    actor Driver
    participant App as Aarokya SDK
    participant API as Aarokya Backend
    participant TC as Transcorp

    Driver->>App: Enters wallet info (if needed)
    App->>API: POST /wallet
    API->>TC: Register customer + create wallet
    TC-->>API: wallet_id, status
    API-->>App: Wallet created
    App->>Driver: Show wallet screen
```

**APIs called:**

| API | Method | Purpose |
|-----|--------|---------|
| `POST /wallet` | Create | Create health savings wallet (Transcorp registration) |

### 9. Wallet & Transactions (Home)

Main wallet view showing balance, recent transactions, and add money option.

```mermaid
sequenceDiagram
    actor Driver
    participant App as Aarokya SDK
    participant API as Aarokya Backend

    App->>API: GET /wallet/{id}/balance
    API-->>App: Current balance
    App->>API: GET /wallet/{id}/transactions/history/30
    API-->>App: Recent transactions
    Driver->>App: Views balance and history
```

**Screen layout:**
- Balance display (prominent)
- Transaction history list
- "Add Money" button
- Each transaction: amount, status, date, description

**APIs called:**

| API | Method | Purpose |
|-----|--------|---------|
| `GET /wallet/{id}/balance` | Read | Fetch live balance from wallet provider |
| `GET /wallet/{id}/transactions/history/{days}` | Read | Fetch transaction history |
